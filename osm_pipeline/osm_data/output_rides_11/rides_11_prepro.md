# output_rides_11 전처리 문서

## 1. 개요

`output_rides_11`은 FrodoBots-2K 데이터셋의 원시(raw) 라이드 데이터 중 하나로,
총 981개의 라이드로 구성되어 있다.

이 문서는 해당 데이터셋에서 학습에 적합한 라이드를 선별하고,
`ride0`(Arrow 기반 파이프라인)과 동일한 포맷의 odom/OSM 맵 이미지를 생성하는
전처리 과정 전체를 기록한다.

---

## 2. 데이터 구조

### 2.1 원시 데이터 구조 (라이드별)

```
output_rides_11/
  ride_XXXXX_YYYYMMDDHHMMSS/
    gps_data_XXXXX.csv              # GPS 궤적 (1Hz, lat/lon/timestamp)
    front_camera_timestamps_XXXXX.csv  # 카메라 프레임 타임스탬프 (20Hz)
    control_data_XXXXX.csv          # 제어 입력 데이터
    imu_data_XXXXX.csv              # IMU 데이터
    recordings/                     # 비디오 클립 (.ts 형식)
```

### 2.2 GPS 특성

| 항목 | 값 |
|---|---|
| GPS 샘플링 레이트 | **1Hz** (초당 1포인트) |
| 카메라 프레임레이트 | **20Hz** (초당 20프레임) |
| GPS 해상도 | 소수점 5~6자리 (약 1m 정밀도) |
| 타임스탬프 단위 | 밀리초 (ms) |

GPS는 1Hz이므로, odom 맵 이미지는 **GPS 포인트 1개당 1장** 생성된다
(카메라 프레임이 아닌 GPS 포인트 기준).

## 4. odom/OSM 맵 이미지 생성 방법

### 4.1 ride0와의 비교

| 항목 | ride0 (Arrow 파이프라인) | output_rides_11 |
|---|---|---|
| 데이터 형식 | Apache Arrow (.arrow) | 원시 GPS CSV + .ts 비디오 |
| 세그먼트 분할 | `episode_selector.py` (정지 구간 제거) | 라이드 단위 직접 처리 |
| 프레임 기준 | 카메라 프레임 (~10Hz) | **GPS 포인트 (1Hz)** |
| 헤딩 정보 | `filtered_heading` (EKF) | **GPS 궤적 기반 진행 방향 추정** |
| 출력 단위 | `episode_XXXX_segYY/` | `ride_XXXXX/` |

### 4.2 헤딩 추정 방법

`filtered_heading`이 없으므로 GPS 궤적에서 직접 진행 방향을 추정한다.

```
현재 위치 → 다음 위치 벡터의 방향각을 헤딩으로 사용
heading_rad = atan2(dlat, dlon * cos(lat))  # East=0, North=π/2
```

마지막 포인트는 직전 포인트의 헤딩을 그대로 사용한다.

### 4.3 OSM 맵 이미지 생성 파라미터

ride0와 동일한 파라미터를 사용한다:

| 파라미터 | 값 |
|---|---|
| 출력 이미지 크기 | **224 × 224 px** |
| Zoom 레벨 | **18** (~0.6m/px) |
| Ego 반경 (map_range_m) | **25m** |
| 과거 경로 색상 | **회색** `(160, 160, 160)` |
| 미래 경로 색상 | **빨간색** `(0, 0, 255)` BGR |
| 헤딩 회전 공식 | `rot_deg = 90 - heading_deg` (ego-heading-up) |
| Ego 마커 | 초록 원 `(0, 200, 0)` |

### 4.4 경로 처리 방법

1. 라이드의 GPS CSV를 읽어 위도·경도·타임스탬프 배열 로드
2. 연속 중복 GPS 좌표 제거 (GPS 해상도 한계 대응)
3. 1m 간격으로 densify (선형 보간)
4. 각 GPS 포인트에 대해 헤딩 추정
5. 현재 포인트 기준 과거/미래 경로 분리
6. `render_frame()` 호출 → 224×224 ego-heading-up 이미지 생성
7. BEV 조감도 1장 생성 (north-up, 시작점 중심, 거리 기반 zoom)

## 6. 학습 데이터 활용 방법

생성된 odom/OSM 맵 이미지는 ride0와 동일한 방식으로 모델 입력으로 사용된다.

```
모델 입력:
  - 카메라 이미지 (front_jpg 또는 .ts 비디오에서 추출)
  - odom/OSM 맵 이미지 (osm_map_XXXXXX.png)

모델 출력:
  - 현재 위치에서의 이동 명령 (속도, 조향)
```

GPS 1Hz 기준이므로, 카메라 프레임(20Hz)과 매칭 시
**타임스탬프 기반 nearest-neighbor 보간**으로 각 카메라 프레임에
가장 가까운 GPS 포인트의 odom 맵을 대응시킨다.
